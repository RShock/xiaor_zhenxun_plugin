# handle2 - 拆字猜成语

## 游戏规则
- 发送"猜成语"开始游戏，10次机会猜一个四字成语
- 每个汉字按一级拆分分为部件，部件独立匹配
- 青色=部件存在且位置正确，橙色=部件存在但位置不正确，灰色=部件不存在
- 独体字不可拆分，整字作为一个部件匹配
- 防作弊：若答案中没有超过20画的字，使用20画以上的字将被拒绝

## 提示系统
- 初始：揭示所有一画部件；若没有一画部件则揭示1个笔画最少部件；再额外揭示1个未揭示的最少笔画部件
- 渐进：第3次失败后揭示1个新部件，第5次失败后再揭示1个
- 每次猜测（无论对错）都会额外揭示1个新部件
- 主动提示：发送"提示"消耗1次机会并揭示1个新部件（提示=猜一次）

## 奖励系统
- 基础奖励400金币，每多猜错一次衰减25%（×0.75）

## 字形拆分系统
- 基于 CJKVI-IDS 数据，解析 Unicode IDS 运算符（⿰⿱⿲⿳⿴⿵⿶⿷⿸⿹⿺⿻）
- 一级拆分：只拆分一层结构，非单字子节点递归展开为叶子部件
- 独体字判断：IDS等于自身 / ⿻叠加且部件笔画≤3
- 包围结构绘制：外部件占对应边角/边缘，内部结构保留IDS递归并占满剩余主区域
- 过滤含编号标记（①②等）的IDS条目

## 匹配算法
- 全局部件匹配：跨4个字位置进行Wordle式匹配
- 优先级：整字完全匹配 > 同位置部件correct > 跨位置部件exist
- 整字状态仅取决于整字是否出现在答案中，与部件匹配无关

## 流程注意
- 游戏记录在开局时创建，用于每日次数统计
- 正常胜负、抢答胜负、主动结束、超时都必须先保存 `is_win/attempts`，再清理内存状态

## 文件结构
- `__init__.py` - nonebot插件入口，命令注册与游戏流程
- `data_source.py` - 兼容导出层，保留旧导入路径
- `game.py` - 游戏核心状态、提示、匹配和状态序列化
- `ids.py` - IDS树结构、解析器、IDS/笔画数据解析函数
- `renderer.py` - PIL图片渲染与字体管理
- `resources.py` - 资源路径与IDS/笔画数据懒加载
- `config.py` - 插件配置
- `models.py` - 数据库模型（Handle2Record）
- `utils.py` - 工具函数（答案库缓存、random_idiom）
- `game_server.py` - 弱化的Web测试服务器，调用核心状态接口
- `game.html` - Web测试前端
- `resources/data/ids.txt` - IDS数据
- `resources/data/Unihan.zip` - Unihan笔画数据
- `resources/data/answers.json` - 成语答案库
- `resources/fonts/` - 字体文件（HanaMinA/B + NotoSerifSC）
